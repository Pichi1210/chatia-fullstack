# Screenshots checklist

Usa esta lista para tomar capturas reales del sistema funcionando. Antes de guardar cada imagen, verifica que no aparezcan credenciales, tokens, claves API, contrasenas, datos personales reales ni contenido sensible.

- [ ] Abrir el frontend del sistema.
- [ ] Capturar la pagina principal o vista inicial del chatbot.
- [ ] Iniciar una conversacion con el chatbot medico.
- [ ] Escribir un sintoma o necesidad medica en lenguaje natural.
- [ ] Mostrar la deteccion de intencion y entidades en Rasa NLU, si aplica.
- [ ] Mostrar las preguntas de triaje generadas por el sistema.
- [ ] Responder el triaje hasta obtener la recomendacion final.
- [ ] Mostrar la recomendacion final de institucion medica.
- [ ] Capturar una tarjeta o ficha de centro medico recomendado.
- [ ] Abrir logs del backend y capturar una interaccion relevante.
- [ ] Abrir logs de Rasa y capturar el procesamiento de mensajes, si Rasa esta activo.
- [ ] Mostrar contenedores Docker activos.
- [ ] Mostrar una respuesta del endpoint en navegador, Postman o Swagger.
- [ ] Mostrar el despliegue en Coolify si aplica.
- [ ] Guardar cada captura con el nombre sugerido en `docs/screenshots/README.md`.
- [ ] Revisar cada imagen antes de versionarla para confirmar que no contiene informacion sensible.
